D061.查询至少有一门课与学号为”01”的同学所学相同课程的同学的信息

use school;
#法一
select distinct s.* from student s
left join sc on sc.sid=s.sid
where s.sid!=01
and sc.cid in (01,02,03);

#法二
select distinct s.* from student s,sc
where s.sid=sc.sid 
and sc.cid in (
select cid from sc where sc.sid=01)
and s.sid!=01;

D062.查询两门及其以上不及格课程的同学的学号，姓名及其平均成绩

use school;
select s.sid,s.sname,avg(sc.score) as "平均成绩" from student s,sc
where sc.sid=s.sid
and s.sid in (
select sc.sid from sc 
where sc.score<60 group by sc.sid having count(sc.cid)>=2)
group by s.sid,sc.sid;

select s.sid,s.sname,avg(sc.score) as "平均成绩" from student s,sc
where sc.sid=s.sid
and s.sid in (
select sc.sid from sc 
where sc.score<60 group by sc.sid having count(1)>=2)
group by s.sid,sc.sid;

D063.检索”01”课程分数小于60，按分数降序排列的学生信息

use school;

and sc.score<60 
and sc.cid=01 group by sc.sid
order by sc.score desc;

D080.查询各科成绩前三名的学生编号，课程编号，成绩，名次

use school;
select* from (
select t.* , px = (select count(distinct score) from sc where cid = t.cid and score >= t.score) from sc t) m

D085.统计每门课程的学生选修人数（超过5人的课程才统计）。要求输出课程号和选修人数，查询结果按人数降序排列，若人数相同，按课程号升序排列

use school;

Hexo Next主题评论插件设置

发表于 2016-09-25 | 分类于 BlogBuild | | 阅读次数:
字数统计: 872 | 阅读时长 ≈ 3

本文主要内容是介绍如何在自己的博客网站安装评论系统插件，以便阅读者可以在读完博客后发表评论。本文便以Hexo Next主题博客为例，介绍如何安装畅言和Valine评论系统和管理评论。

一、畅言评论插件

继强大且免费的评论系统多说下线后，畅言算得上比较普遍且易于管理评论数据的评论插件了，但是比较麻烦一点的是畅言评论系统需要备案域名审核，对于利用Github免费空间创建的个人博客网站来说，域名是没有备案的，不过本人亲测一个忽略备案审核设置的方法，会在下面介绍给大家。

Next主题支持多种评论插件，所以设置起来非常简单，主要就是注册畅言评论系统并获取你的账号APPid和APPkey。

1.注册畅言账号

首先用手机号码在畅言官网进行注册（是注册畅言而不是快站哦），注意在填写站点名称、站点网站和ICP备案号时，请先找一个比较小众但是会长期使用的网站域名，并在ICP备案号查询网站查询其备案号，接下来在域名白名单中填入你的博客域名地址即可，这样就可以通过审核并使用畅言系统。下图为我注册时的信息（用的是实习公司网站）。

另外，有关那些代码设置不需要去管，跳过即可。

2.配置Next主题设置

注册好畅言账号后，打开后台总览找到畅言秘匙将APPid和APPkey分别复制到Next主题文件目录下的配置文件中（例如这是我的路径:E:\MyBlog\themes\next\_config.yml）的#changyan下方的appid和appkey(大约在第372行)，并且将enable: false修改为enable: true。

配置好以后就可以使用畅言评论系统啦！

二、Valine评论插件

畅言评论系统的优点是易于管理你的评论数据，同时还可以增加其他的评论相关功能，例如印章、热门评论、评论点赞等等，但是需要登录畅言才能够发表评论，有时候游客阅读后可能不太希望必须注册登录才能发表评论，这时候就可一采用Valine评论插件，这是一个无需登录就能发表评论的插件，更为便捷而且免费。

1.注册LeanCloud账号

打开LeanCloud官网,点击免费试用进入注册界面，用github账号、微博账号或者QQ账号注册LeanCloud账号，进入开发者信息页面，点击应用/创建新应用，在创建的应用中打开应用Key,可以找到APPid和APPkey。

2.配置Next主题设置

同样地，在Next主题文件目录下的配置文件中（例如这是我的路径:E:\MyBlog\themes\next\_config.yml）的#valine下方的appid和appkey(大约在第382行)，并且将enable: false修改为enable: true（注意：当你选择使用valine评论时，你应该要将其他评论插件设置为enable: false）。

#最后，大家也可根据自己的喜好，设置其他的评论插件，Next主题有多种评论插件的接口，设置起来也非常简单，建议大家自己动动手设置其他新的插件！

Github+Hexo搭建个人博客

发表于 2016-09-18 | 分类于 BlogBuild | | 阅读次数:
字数统计: 1,073 | 阅读时长 ≈ 4

本文详细地说明了如何运用开源框架Hexo和使用Github免费的云空间完成个人免费博客的搭建，除此之外，还包括许多Hexo主题进行博客个性化设计的步骤．

一、知识了解

1．什么是Hexo

Hexo中文网站是一个快速，简洁且高效的博客框架．让上百个页面在几秒内瞬间完成渲染．Hexo支持Github　Flavored　Markdown的所有功能．甚至可以整合Octopress的大多数插件．并且自己也拥有强大的插件系统．简单理解Hexo就是一个开源的博客设计框架，个人可以随意进行下载、修改和设计．

２．什么是GIt和Github

通俗地解释，git是一种版本控制系统．跟svn、cvs是同级的概念．
github是一个网站，给用户提供git服务．这样你就不用自己部署git系统，直接用注册个账号，用他们提供的git服务就可以．可以理解为Git是在本地电脑上处理各种文件，而Github就像百度云盘云盘一样．更详细的解释可以参见大白话解释 Git 和 GitHub．

3．其他

诸如Node.js等都可以理解为一种程序语言解释器，不需要刻意掌握．

二、配置准备

1．Git

在Git官网下载属于本电脑配置的Git安装包并安装Git，安装过程比较简单，如有问题请参见Git安装配置教程．

2．Github

在Github官网注册Github账号，然后新建一个项目（PS：项目名称必须为:Github账号名.github.io，这也是设计的博客网址）．

3．Node.js

在Node.js官网下载后缀为LTS的安装包并安装．

三、本地搭建过程

1．安装Hexo.打开CMD命令行窗口(Ctrl+R)或者Windows PowerShell.运行如下命令行:

npm install -g hexo-cli

2．新建MyBlog文件夹,文件地址如:E:\MyBlog.并在该文件夹内右键点击Git Bash here.

3．依次输入并运行如下命令行:

hexo init
hexo install
hexo g
hexo s

这时便生成你的本地博客网页：https://localhost:4000，你便可以在浏览器中打开并预览你的博客网页（PS：如果你的电脑安装有福昕阅读器，会占用你的https://localhost:4000地址，此时你可以通过运行
hexo s -p 4001生成本地地址https://localhost:4001来预览你的本地博客。另外，请不要Crtl+C导致`hexo s`停止，否则无法打开你的本地博客网页。）

以上便是如何生成本地博客网页，建议你以后书写博文或者配置博客都先通过hexo g和hexo s两个命令在本地博客网页进行预览(一般修改配置后建议可先执行hexo clean清除缓存)，再部署到你的github上（也就是生成网页域名来查看你的博客网站，接下来便是如何进行部署）。

四、博客部署过程

1．在MyBlog文件夹内右键点击Git Bash here并执行：

npm install hexo-deployer-git --save

2．打开Github中的已新建的项目,打开Clone or download复制项目地址:格式如https://github.com/Github账号名/Github账号名.github.io.git

3．打开MyBlog文件夹中的-config.yml(可以下载一个Notepad++或者其他编译器打开),修改最下方的deploy为:

deploy:
  type: git
  repo: https://github.com/Github账号名/Github账号名.github.io.git
  branch: master

注意有一个空格，修改后保存！！！

4．再运行如下命令行:

hexo g -d

至此,你就可以通过网址:Github账号名+github.io访问属于你自己的博客啦!!!

五、如何写博客文章

1．在Markdown官网下载Markdownpad2并安装

Markdown其实就相当于一个Office Word的一个编辑软件，学习和使用很容易掌握，具体语法可参考Markdown教程。

2．通过Markdownpad书写你的博客并放在MyBlog/source目录下

3．通过hexo g和hexo s在本地https://localhost:4000预览，再通过`hexo d`部署到你的博客网站上去，博客书写完成！！！

一、Next主题安装

打开Next主题下载地址，下载source code压缩包，解压并放至MyBlog\theme文件夹。打开MyBlog目录下的_config.yml，修改其中的theme: landscape为theme: next(注意：此处的名称要与之前解压的文件名一致，一般解压后应该修改文件名的)。

二、风格切换

打开MyBlog/theme/next目录下的_config.yml，Next提供了Muse、Mist、Pisces、Gemini四种风格，具体设置在# Schemes（约在第110行左右）下方，将你需要设置的风格前的#去掉即可，其余的风格前必须都加上#。

三、分享插件使用

分享插件在_config.yml中约在第446行的needmoreshare2处进行设置，将其下的3个enable: false均修改为enable: true(注意空格！)

四、背景切换

背景切换在在_config.yml中约在第697行至707行的canvas_nest、three_waves、canvas_lines、canvas_sphere处进行设置，将你需要的背景设置为true即可。

五、有关标签、分类等菜单栏设置的问题

关于菜单栏设置是在MyBlog/theme/next目录下的_config.yml中的menu（约在第94行）部分，将你需要使用的菜单窗口前的#去掉即可。

六、其他可能遇到的问题及作者建议

  • 可以在MyBlog目录下的_config.yml中的#Site部分设置title、subtitle、description等，其中建议将language设置为language: zh-Hans。

  • 有可能出现菜单栏设置的标签或归类打不开，这时可通过在MyBlog目录下运行hexo new page "tags"或hexo new page "categories"等使得在source目录下生成tags和categories文件夹。

  • 建议在书写博文时在开头完善各个信息以便浏览你的博客，如下书写方式：

    title: Hexo Next主题博客设置
    comments: true
    date: 2018-06-28 22:37:42
    categories: BlogBuild
    tags: [Hexo,Next]
